# backtest

当前包含可信历史验证回测底座：

- `engine.py`：point-in-time 横截面回测。信号使用信号日收盘后可得历史因子，默认下一交易日开盘执行，输出策略、沪深 300、股票池等权和单因子基线。
- `execution.py`：日线级执行近似，处理无量、缺价、涨停难买、跌停难卖和延迟退出。
- `metrics.py`：集中计算累计收益、年化收益、最大回撤、年化波动率、夏普比率、胜率、单期收益、换手率、交易次数、有效期数和相对基准超额收益。

默认命令：

```powershell
python scripts\run_minimal_backtest.py
```

默认历史回测不再使用当天候选 CSV 向前回测，也不再把实时扫描缺失因子动态重配后伪装成同一策略。CSV 股票池和当前快照流动性过滤仅作为显式调试/诊断模式保留。

已知限制：

- 默认股票池仍是当前仍上市 A 股代码池过渡方案，历史退市股票未完整纳入，仍存在幸存者偏差。
- 腾讯历史源成交额由成交手数乘价格派生，需在报告中披露。
- 日线只能近似停牌和涨跌停可成交性，不能替代逐笔盘口验证。
